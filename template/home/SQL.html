1. SQL Introduction

SQL (Structured Query Language) is the standard language for interacting with relational databases. It is used to create, read, update, and delete data.

How to Start with SQL

To start using SQL, you'll need to set up a database management system (DBMS) that supports SQL. Here are some popular options:

  • MySQL: A widely used open-source relational database. You can download it from the MySQL website.
  • PostgreSQL: An advanced open-source relational database known for its robustness. Download it from the PostgreSQL website.
  • SQLite: A lightweight, serverless database that's easy to set up. You can download it from the SQLite website.
  • Microsoft SQL Server: A popular database system for enterprise applications. Download the Express version for free from the Microsoft website.

Setting Up SQL

After downloading a DBMS, follow these general steps to set it up:

  1. Run the installer and follow the prompts to install the software.
  2. Configure the database settings as needed (e.g., setting a root password).
  3. Launch the database management interface (like MySQL Workbench, pgAdmin, etc.) to start interacting with your database.

Basic SQL Commands

Once your database is set up, you can start executing SQL commands. Here are a few basic commands to get you started:

  • CREATE: To create a new table.
  • SELECT: To retrieve data from a table.
  • INSERT: To add new data to a table.
  • UPDATE: To modify existing data in a table.
  • DELETE: To remove data from a table.

2. SQL Syntax

The basic syntax of SQL involves using various commands like SELECT, INSERT, UPDATE, and DELETE to interact with the database.

Common SQL Commands

  • SELECT: Used to select data from a database. 
    SELECT * FROM students;

    '*' -symbol is called the "asterisk," and it is used as a wildcard to indicate "all columns" when querying data from a table

    Output:

    
        id  | name      | age | grade
        -----------------------------
        1   | John Doe  | 20  | A
        2   | Jane Smith| 21  | B
        3   | Alice Brown| 22 | A
  • INSERT: Used to insert new data into a database. 
    INSERT INTO students (name, age, grade) VALUES ('Bob Johnson', 23, 'B');

    Output:

    1 row inserted.
  • UPDATE: Used to modify existing data in a database. 
    UPDATE students SET grade = 'A' WHERE name = 'Jane Smith';

    Output:

    1 row updated.
  • DELETE: Used to delete existing data from a database. 
    DELETE FROM students WHERE name = 'Alice Brown';

    Output:

    1 row deleted.

13. SQL Aggregate Functions

Aggregate functions perform a calculation on a set of values and return a single value.

  • COUNT: Returns the number of rows.
  • SUM: Returns the total sum of a numeric column.
  • AVG: Returns the average value of a numeric column.
  • MIN: Returns the smallest value.
  • MAX: Returns the largest value.

Examples

COUNT Example

SELECT COUNT(*) FROM Customers;

Output:

10

SUM Example

SELECT SUM(OrderAmount) FROM Orders;

Output:

5000.00

AVG Example

SELECT AVG(OrderAmount) FROM Orders;

Output:

500.00

MIN Example

SELECT MIN(OrderAmount) FROM Orders;

Output:

100.00

MAX Example

SELECT MAX(OrderAmount) FROM Orders;

Output:

1000.00

18. SQL LIKE

The LIKE operator is used to search for a specified pattern in a column. It allows for partial matching of strings using wildcards.

Wildcards Used with LIKE

1. % (Percent Sign) - Represents zero, one, or multiple characters.

SELECT * FROM Customers WHERE CustomerName LIKE 'a%';

This query retrieves all customers whose names start with 'a'. For example, 'Andrew', 'Alice', and 'Amanda'.

2. _ (Underscore) - Represents a single character.

SELECT * FROM Customers WHERE CustomerName LIKE '_n%';

This query retrieves all customers whose name has 'n' as the second character. For example, 'Andrew' or 'Bend'.

3. [] (Square Brackets) - Matches any single character within the brackets.

SELECT * FROM Customers WHERE CustomerName LIKE '[A-C]%';

This query retrieves all customers whose names start with a letter from A to C.

4. [^] (Caret in Brackets) - Matches any single character not in the brackets.

SELECT * FROM Customers WHERE CustomerName LIKE '[^A-C]%';

This query retrieves all customers whose names do not start with A, B, or C.

Examples

Example 1: Searching for names that start with 'a':

SELECT * FROM Customers WHERE CustomerName LIKE 'a%';

Output:


            CustomerID | CustomerName
            --------------------------
            1          | Andrew
            2          | Alice
            3          | Amanda

Example 2: Searching for names that contain 'and':

SELECT * FROM Customers WHERE CustomerName LIKE '%and%';

Output:


            CustomerID | CustomerName
            --------------------------
            1          | Andrew
            2          | Amanda

Example 3: Searching for names that end with 'a':

SELECT * FROM Customers WHERE CustomerName LIKE '%a';

Output:


            CustomerID | CustomerName
            --------------------------
            1          | Amanda
            2          | Anna

Case Sensitivity

By default, the LIKE operator is case-insensitive in most SQL databases (such as MySQL). However, it can be case-sensitive depending on the database collation settings (like in SQL Server).

Performance Considerations

Using the LIKE operator with leading wildcards (e.g., '%pattern%') may result in slow queries, as it doesn't take advantage of indexes.

19. SQL Wildcards

Wildcards are special characters used with the LIKE operator to substitute for unknown characters in SQL queries. The two most common wildcards are:


            % - Represents zero or more characters
            _ - Represents a single character

Examples of Using Wildcards

Example 1: Using `%` to match any string that starts with 'A'.

SELECT * FROM Customers WHERE CustomerName LIKE 'A%';

Output:


            CustomerID | CustomerName
            --------------------------
            1          | Andrew
            2          | Alice
            3          | Amanda
            4          | Angela

Example 2: Using `_` to match any string where the second letter is 'n'.

SELECT * FROM Customers WHERE CustomerName LIKE '_n%';

Output:


            CustomerID | CustomerName
            --------------------------
            1          | Andrew
            2          | Ben
            3          | Tony

Example 3: Using `%` at the beginning to find customers whose names end with 'a'.

SELECT * FROM Customers WHERE CustomerName LIKE '%a';

Output:


            CustomerID | CustomerName
            --------------------------
            1          | Amanda
            2          | Sophia

Example 4: Using `_` to match any name where the first character is 'A' and the second character is 'n'.

SELECT * FROM Customers WHERE CustomerName LIKE 'An_';

Output:


            CustomerID | CustomerName
            --------------------------
            1          | Andy
            2          | Ann

Example 5: Using square brackets to match a character within a range. Find names starting with 'A', 'B', or 'C'.

SELECT * FROM Customers WHERE CustomerName LIKE '[A-C]%';

Output:


            CustomerID | CustomerName
            --------------------------
            1          | Alice
            2          | Bob
            3          | Charlie

Example 6: Using `[^]` to match names that do not start with 'A', 'B', or 'C'.

SELECT * FROM Customers WHERE CustomerName LIKE '[^A-C]%';

Output:


            CustomerID | CustomerName
            --------------------------
            1          | David
            2          | Emily

23. SQL Joins

Joins are used to combine rows from two or more tables based on a related column. The different types of joins are:

  • INNER JOIN: Returns records that have matching values in both tables.
  • LEFT JOIN: Returns all records from the left table and the matched records from the right table.
  • RIGHT JOIN: Returns all records from the right table and the matched records from the left table.
  • FULL JOIN: Returns all records when there is a match in either left or right table records.
  • SELF JOIN: Joins a table with itself.

Example 1: INNER JOIN

SELECT Orders.OrderID, Customers.CustomerName
            FROM Orders
            INNER JOIN Customers ON Orders.CustomerID = Customers.CustomerID;

Output:


            OrderID | CustomerName
            -----------------------
            1       | John Doe
            2       | Jane Smith
            3       | Mike Johnson

Example 2: LEFT JOIN

SELECT Customers.CustomerName, Orders.OrderID
            FROM Customers
            LEFT JOIN Orders ON Customers.CustomerID = Orders.CustomerID;

Output:


            CustomerName | OrderID
            ----------------------
            John Doe     | 1
            Jane Smith   | 2
            Mike Johnson | 3
            Emily Davis  | NULL

Explanation: In this example, all customers are listed, even if they don't have any orders. The missing order for "Emily Davis" is represented by a NULL value.

Example 3: RIGHT JOIN

SELECT Orders.OrderID, Customers.CustomerName
            FROM Orders
            RIGHT JOIN Customers ON Orders.CustomerID = Customers.CustomerID;

Output:


            OrderID | CustomerName
            -----------------------
            1       | John Doe
            2       | Jane Smith
            3       | Mike Johnson
            NULL    | Emily Davis

Explanation: The RIGHT JOIN returns all orders and their matching customers, even if some customers don't have orders.

Example 4: FULL JOIN

SELECT Orders.OrderID, Customers.CustomerName
            FROM Orders
            FULL JOIN Customers ON Orders.CustomerID = Customers.CustomerID;

Output:


            OrderID | CustomerName
            -----------------------
            1       | John Doe
            2       | Jane Smith
            3       | Mike Johnson
            NULL    | Emily Davis

Explanation: The FULL JOIN returns all records when there is a match in either the left (Orders) or right (Customers) table. It includes rows where there is no match in either table.

Example 5: SELF JOIN

SELECT A.CustomerName AS Customer, B.CustomerName AS Referrer
            FROM Customers A, Customers B
            WHERE A.ReferrerID = B.CustomerID;

Output:


            Customer    | Referrer
            ------------------------
            John Doe    | Jane Smith
            Mike Johnson| John Doe

Explanation: In a SELF JOIN, the table is joined with itself. In this example, customers who were referred by others are listed with the referrer name.

25. SQL GROUP BY

The GROUP BY statement groups rows that have the same values in specified columns into summary rows.

Example

SELECT COUNT(*), Country FROM Customers
        GROUP BY Country;

Output:


        COUNT(*) | Country
        --------- | --------
        5        | Germany
        3        | France
        7        | USA

26. SQL HAVING

The HAVING clause was added to SQL because the WHERE keyword could not be used with aggregate functions.

Example

SELECT COUNT(*), Country FROM Customers
        GROUP BY Country HAVING COUNT(*) > 5;

Output:


        COUNT(*) | Country
        --------- | --------
        8        | Germany

27. SQL EXISTS

The EXISTS operator is used to check for the existence of any record in a subquery.

Example

SELECT * FROM Customers
        WHERE EXISTS (SELECT * FROM Orders
                       WHERE Customers.CustomerID = Orders.CustomerID);

Output:


        CustomerID | CustomerName | Country
        ----------- | ------------ | --------
        1           | Cardinal     | Germany
        2           | Blarine      | France

28. SQL ANY, ALL

ANY and ALL are used to compare a value to a list of values returned by a subquery.

Example

SELECT * FROM Products
        WHERE Price > ANY (SELECT Price FROM Products WHERE Price < 20);

This query retrieves all products that have a price greater than any price from the products that are less than 20.

29. SQL SELECT INTO

The SELECT INTO statement is used to select data from one table and insert it into another table.

Example

SELECT * INTO NewTable FROM Customers WHERE Country='Germany';

This query creates a new table named NewTable and populates it with all records from the Customers table where the country is Germany.

30. SQL INSERT INTO SELECT

The INSERT INTO SELECT statement is used to insert data from one table into another.

Example

INSERT INTO NewTable (CustomerName, Country) SELECT CustomerName, Country FROM Customers WHERE Country='Germany';

This query inserts the CustomerName and Country from the Customers table into the NewTable for all customers from Germany.

31. SQL CASE

The CASE statement goes through conditions and returns a value when the first condition is met.

Example

SELECT CustomerName, CASE WHEN Country='Germany' THEN 'EU' ELSE 'Non-EU' END AS Region FROM Customers;

This query selects the CustomerName and categorizes customers based on their Country, assigning them to either 'EU' or 'Non-EU' regions.

32. SQL NULL Functions

NULL functions are used to handle NULL values in a database. These functions allow you to replace NULLs with a default value or perform operations that ignore NULLs.

Example

SELECT IFNULL(column_name, 'default_value') AS column_name FROM table_name;

This query uses the IFNULL function to return 'default_value' when column_name is NULL.

33. SQL Stored Procedures

Stored procedures are prepared SQL code that you can save and reuse. They help in encapsulating the SQL logic and can be executed with a simple call.

Example

CREATE PROCEDURE ProcedureName AS
        BEGIN
            SELECT * FROM Customers;
        END;

This example creates a stored procedure named ProcedureName that retrieves all records from the Customers table. You can execute this stored procedure using the following command:

EXEC ProcedureName;

This will run the stored procedure and return all records from the Customers table.

34. SQL Comments

Comments in SQL are used to make the code more understandable and to provide explanations or annotations for future reference. They are ignored by the SQL engine.

Example

-- This is a single-line comment
/* This is a multi-line comment */

In the example above:

  • Single-line comments start with -- and continue to the end of the line.
  • Multi-line comments are enclosed between /* and */ and can span multiple lines.

35. SQL Operators

SQL operators are used to specify conditions in a SQL statement. They help in filtering records and performing operations on data.

Types of SQL Operators:

  • Comparison Operators:
    • = : Equal to
    • < : Less than
    • > : Greater than
    • <= : Less than or equal to
    • >= : Greater than or equal to
    • <> : Not equal to
  • Arithmetic Operators:
    • + : Addition
    • - : Subtraction
    • * : Multiplication
    • / : Division
  • Logical Operators:
    • AND : Returns true if both conditions are true
    • OR : Returns true if at least one condition is true
    • NOT : Reverses the result of a condition
